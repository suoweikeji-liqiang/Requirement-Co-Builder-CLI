<task type="auto" tdd="true">
  <name>Task 1: Atomic write layer + Project state schema</name>
  <files>src/state/atomic.ts, src/state/schema.ts</files>
  <behavior>
    atomicWrite(filePath, data):
    - Writes data to filePath + '.tmp' in same directory as filePath (NOT os.tmpdir() — avoids EXDEV)
    - Calls fs.promises.rename(tmpPath, filePath)
    - If rename throws with code 'EXDEV': calls fs.promises.copyFile(tmpPath, filePath) then fs.promises.unlink(tmpPath)
    - If any other error: rethrows it

    readStateWithRecovery(filePath):
    - Tries fs.promises.readFile(filePath, 'utf8') — returns content on success
    - If ENOENT or read fails: tries to read filePath + '.tmp'
    - If .tmp exists: renames .tmp to filePath (promotes recovery file), returns content
    - If .tmp also missing: returns null

    ProjectStateSchema:
    - id: z.string()
    - idea: z.string()
    - clarityStage: z.enum(['concept', 'direction', 'structure', 'executable'])
    - messages: z.array(MessageSchema) — MessageSchema has role (z.enum(['user','assistant','system'])), content (z.string()), timestamp (z.string())
    - createdAt: z.string()
    - updatedAt: z.string()
    - useLocal: z.boolean()
    - Export type ProjectState = z.infer<typeof ProjectStateSchema>
    - Export type Message = z.infer<typeof MessageSchema>
  </behavior>
  <action>
    Create src/state/atomic.ts implementing atomicWrite and readStateWithRecovery. Follow the exact patterns from the research document (Pattern 3: Atomic State Write). Key points:
    - Write temp file to the SAME directory as destination: use `filePath + '.tmp'` not `path.join(os.tmpdir(), ...)`
    - The EXDEV error code is 'EXDEV' — check (err as NodeJS.ErrnoException).code === 'EXDEV'
    - On EXDEV: copyFile then unlink (not another rename attempt)
    - readStateWithRecovery: if state.json is missing/corrupt, promote .tmp by renaming it to the main path before returning

    Create src/state/schema.ts defining:
    - MessageSchema with role/content/timestamp fields
    - ProjectStateSchema with all project state fields
    - Exported types: Message, ProjectState
    - Keep schemas flat — no recursive references (avoid Zod v4/AI SDK compatibility issues noted in research pitfall 4)

    Both files must use Node.js built-in 'node:fs' and 'node:path' imports (no extra deps).
    All exports use named exports (no default exports) — consistent ESM pattern.
  </action>

<task type="auto" tdd="true">
  <name>Task 2: State read/write functions + Project ID generator + Error display</name>
  <files>src/state/index.ts, src/utils/id.ts, src/display/errors.ts</files>
  <behavior>
    readProjectState(projectDir):
    - Calls readStateWithRecovery(path.join(projectDir, 'state.json'))
    - Returns null if raw is null
    - Parses JSON, runs ProjectStateSchema.parse()
    - On Zod parse error: returns null (treat corrupt state as missing, let caller recover)

    writeProjectState(projectDir, state):
    - Sets state.updatedAt = new Date().toISOString()
    - Calls atomicWrite(path.join(projectDir, 'state.json'), JSON.stringify(state, null, 2))

    createInitialState(id, idea, useLocal):
    - Returns a valid ProjectState with clarityStage='concept', messages=[], timestamps set to now

    generateProjectId(idea):
    - Creates 4-char hex from SHA-256 of (idea + Date.now()) — crypto.createHash('sha256').update(idea + Date.now()).digest('hex').slice(0, 4)
    - Creates slug: lowercase, strip non-alphanumeric (keep spaces), trim, split words, take first 5, join with '-'
    - Input 'Build a CLI tool' → something like 'a3f2-build-a-cli-tool'
    - Input with Chinese characters: strip non-ASCII for slug (Chinese chars become empty after strip), use 'idea' as fallback slug if slug is empty
    - Returns `${hash}-${slug}` or just hash if slug is empty

    formatError(err):
    - If err is Error: returns err.message
    - If string: returns err
    - Otherwise: returns 'An unexpected error occurred'
    - Never returns stack traces in user-facing output

    printError(err):
    - Calls chalk.red('Error: ' + formatError(err)) and writes to stderr via process.stderr.write
  </behavior>
  <action>

    - Import atomicWrite, readStateWithRecovery from './atomic.js'

    - Implement readProjectState, writeProjectState, createInitialState
    - readProjectState: handles both null raw and Zod parse failure gracefully (return null in both cases)
    - writeProjectState: always updates updatedAt before writing — produce new state object (immutability pattern per coding style rules)

    Create src/utils/id.ts:
    - Import crypto from 'node:crypto'
    - Implement generateProjectId(idea: string): string
    - Hash: sha256 of (idea + Date.now()), take first 4 hex chars
    - Slug: idea.toLowerCase().replace(/[^a-z0-9\s]/g, '').trim().split(/\s+/).filter(Boolean).slice(0, 5).join('-')
    - If slug is empty (e.g., idea was all Chinese): use 'project' as fallback
    - Return `${hash}-${slug || 'project'}`

    - Import chalk from 'chalk' (ESM v5)

    - Implement printError(err: unknown): void — writes to stderr, not stdout
    - Follow error handling rules: user-friendly messages, no stack traces in output
