# Requirement Co-Builder CLI 使用文档

## 1. 简介

`Requirement Co-Builder CLI` 是一个命令行需求共创工具，用于把模糊想法逐步沉淀为可执行输出文件。

当前版本支持：
- 项目创建、查看、删除、列表
- 对话续聊
- 快照归档
- 需求构建输出（`spec.md` / `acceptance.md` / `tasks.md`）
- 研究记录（note / link）

## 2. 环境要求

- Node.js `>= 18`
- npm（随 Node.js 安装）

## 3. 安装与运行

在项目根目录执行：

```bash
npm install
```

开发模式运行（直接执行 TS 入口）：

```bash
npm run dev -- --help
```

构建后运行：

```bash
npm run build
node dist/bin/req.js --help
```

如果希望在当前机器全局用 `req` 调用，可在项目目录执行：

```bash
npm link
req --help
```

## 4. 首次配置

先设置 API Key 与默认 Provider：

```bash
req config set-key openai <YOUR_OPENAI_KEY>
req config set-provider openai
```

或使用 Anthropic：

```bash
req config set-key anthropic <YOUR_ANTHROPIC_KEY>
req config set-provider anthropic
```

支持的 provider 仅有：
- `openai`
- `anthropic`

## 5. 存储模式

CLI 支持两种存储位置：

- 默认：`~/.reqgen/projects/`
- 本地模式：加全局参数 `--local`，存到当前目录 `./projects/`

示例：

```bash
req --local new "做一个客服工单系统"
```

## 6. 命令总览

### 6.1 项目管理

创建项目：

```bash
req new "<idea>"
```

列出项目：

```bash
req list
```

查看项目摘要：

```bash
req open <projectId>
```

删除项目：

```bash
req delete <projectId>
```

### 6.2 对话与快照

续聊：

```bash
req chat <projectId>
```

会话中可输入：
- `/exit` 或 `/quit` 结束会话

创建快照：

```bash
req snapshot <projectId> --tag v0.1
```

不传 `--tag` 时使用时间戳；重复 tag 会报错。

### 6.3 构建输出

把当前项目状态编译为输出文件：

```bash
req build <projectId>
```

会生成：
- `spec.md`
- `acceptance.md`
- `tasks.md`

### 6.4 研究记录

追加研究笔记：

```bash
req research add-note <projectId> <note...>
```

追加研究链接：

```bash
req research add-link <projectId> <url> --title "<title>"
```

两者都会写入项目下的 `research.md`。

## 7. 典型流程

```bash
# 1) 配置 provider（仅首次）
req config set-key openai <KEY>
req config set-provider openai

# 2) 创建项目
req --local new "做一个面向中小团队的排班系统"

# 3) 查看项目列表，拿到 projectId
req --local list

# 4) 进入对话
req --local chat <projectId>

# 5) 需要时保存快照
req --local snapshot <projectId> --tag first-draft

# 6) 构建输出
req --local build <projectId>

# 7) 记录研究信息
req --local research add-note <projectId> "竞品A支持移动端排班"
req --local research add-link <projectId> https://example.com --title "竞品调研"
```

## 8. 产物结构（示例）

单个项目目录下常见文件：

- `state.json`：核心状态
- `idea.md`：每轮对话后的概要同步
- `decisions.md`：确认/修改类决策日志
- `research.md`：研究笔记与链接
- `spec.md` / `acceptance.md` / `tasks.md`：构建输出
- `snapshots/<tag>/`：快照目录

## 9. 常见问题

### Q1：报错 `Project not found`

检查：
- `projectId` 是否正确
- 是否忘记了 `--local`（存储位置不一致时会找不到）

### Q2：`req research add-link` 报参数错误

`add-link` 必须带 `--title`：

```bash
req research add-link <id> <url> --title "标题"
```

### Q3：为什么没生成 `decisions.md`

`decisions.md` 只在检测到明确“确认/修改/决定”等信号时追加；普通问答不会写入。
